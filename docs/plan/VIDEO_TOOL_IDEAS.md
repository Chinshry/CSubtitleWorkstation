# 视频处理工具页候选功能

本文整理来自 `E:\Downloads\@video_tool.bat` 和 `E:\BaiduSyncdisk\BAT` 的视频处理脚本能力，用于评估哪些功能适合放入 CSubtitleWorkstation 的「工具」页。

## 设计边界

工具页不应该重复压制页和 Aegisub 已经覆盖得很好的能力。

- 压制相关：字幕压制、LOGO、水印、编码器、CRF、码率、帧率、反交错等优先留在压制页。
- 字幕编辑相关：时间轴、样式、格式转换、字幕清理、拆分合并等优先交给 Aegisub。
- 工具页适合：不进入完整压制流程、10 秒内能解决明确问题、主要是封装或音视频流处理的功能。

## 工具页结构规划

工具页建议改成两级结构：

1. 一级分组：`文字处理`、`媒体处理`
2. 二级页面：每个页面承载一组目标一致的操作，避免一个 tab 里塞太多互不相关的按钮。

### 页面拆分原则

- 一个页面只解决一类输入输出关系。
- 需要同一种输入的功能可以合并；例如提取音频和提取无声视频都只需要一个视频文件。
- 需要不同输入数量的功能应在页面内明确分区；例如音视频合并需要一个视频文件和一个音频文件。
- 会触发重新编码的功能暂缓，避免和压制页混淆。
- 字幕编辑器已经高质量覆盖的功能不进第一批。

## 脚本能力归类

### 1. 转码和画面处理

来源：`@video_tool.bat`

- 旋转 / 镜像：`transpose`、`hflip`、`vflip`
- 分辨率调整：4K、1080、720、竖屏 1080、竖屏 720、自定义
- 帧率调整：30fps、自定义 fps
- 码率调整：8000k、6500k、4000k、800k、自定义
- 反交错：`yadif`
- 仅重新编码：`libx264 -crf 18`

判断：不建议放工具页。这些本质是无字幕压制/快速转码，和现有压制页职责重叠。

当前实现状态：已按“压制页能力”落地，不新增工具页入口。

- 压制页新增 `视频处理` 区块，可单独启用。
- 旋转与镜像拆成两个配置：旋转支持不旋转、顺时针 90°、逆时针 90°、旋转 180°；镜像支持不镜像、横向镜像、竖向镜像。
- 分辨率支持横屏 4K / 1080 / 720、竖屏 1080 / 720 和自定义 `scale=宽:高`。
- 帧率和视频码率支持留空不调整；帧率输入显示 `fps`，视频码率输入显示 `Kbps`。
- 质量值允许留空；留空时不生成 `-crf` / `-cq` / `-qp` 参数。
- 反交错、LOGO、字幕、AVS 与视频处理共用同一个压制任务流程和命令预览。
- 初始空状态已调整为仅显示全屏拖拽入口，导入视频后再显示完整压制控件。

### 2. 封装转换和流复制

来源：`合并分离转换`

- 任意视频转 MP4 封装：`-vcodec copy -acodec copy`
- TS 合并后转 MP4
- VOB 转 MP4
- MP4 拼接
- M3U8 / TS 分片合并

判断：适合工具页。多数操作可以不重编码，和字幕压制主流程不冲突。

当前实现状态：已落地为 `媒体处理 / 封装转换`。
- 单文件转 MP4：使用 `-map 0 -c copy` 保留所有流；TS / M2TS / MTS 输入自动追加 `-bsf:a aac_adtstoasc`。
- TS 分片合并转 MP4：选择分片目录，按自然排序预览 `.ts` / `.m2ts` / `.mts` 文件，再用 ffmpeg concat demuxer 合并输出 MP4。
- 添加视频封面：选择 MP4 / M4V / MOV 与 JPG / PNG 封面图，使用 `-disposition:v:1 attached_pic` 写入封面流，不重新编码视频和音频。
- 合并视频+音频：选择视频文件与外部音频，使用 `-map 0:v:0 -map 1:a:0 -c copy -shortest` 输出 MP4。
- 失败时不自动重编码；若源流不兼容 MP4 容器，提示用户回到压制页重新编码。

### 3. 音视频分离和合并

来源：`合并分离转换`

- 提取音频：输出 `.m4a`
- 提取无声视频：输出 `.mp4`
- 合并 `.mp4 + .m4a`
- 从 MKV 分离字幕、音频、视频

判断：适合工具页。该类操作是常见的素材整理需求，Aegisub 不覆盖，压制页也不是最自然入口。

### 4. 视频分割

来源：`其他\视频分割N秒.bat`、`其他\视频分割N份.bat`

- 按 N 秒切分
- 按 N 份切分
- 截取固定时间段

判断：可作为第二批功能。它有明确价值，但交互和输出命名需要比脚本更严谨，优先级低于封装/分离/合并。

### 5. 水印

来源：`水印`

- 文字水印
- 图片 LOGO 水印
- 720P / 1080P / 4K 尺寸预设

判断：暂不建议放工具页。现有压制页已有 LOGO 能力，单独放工具页会制造重复入口。除非后续明确做「无字幕加水印」。

### 6. 社交媒体专用

来源：`社交媒体专用`

- 修改 MD5：复制文件后追加随机数据
- VTT 转 ASS
- IG / YouTube 合并脚本

判断：暂不建议放工具页。修改 MD5 不适合作为主功能；VTT 转 ASS 偏字幕编辑，Aegisub 可处理；平台专用合并脚本适合保留为个人脚本而不是通用产品能力。

### 7. CC 字幕整理

来源：`C:\Users\CC\AppData\Roaming\Aegisub\automation\autoload\CC的工具集v1.0.11.lua`

相关宏：

- `格式化CC字幕`
- `替换CC字幕成员名字`

核心逻辑：

- 将 Web CC 里类似 `[ZHANG HAO]Hello...\Nworld` 的文本拆成两行：
  - 原行保留说话人标签，去掉 `[` / `]`，样式改为 `花字`
  - 新插入一行台词，样式使用 `听轴`
- 台词清理：
  - 去掉 `...`
  - 行首 `\N` 转为空格或清除
  - 普通 `\N` 转为空格
  - 多个空格压缩成一个空格
  - 去掉行首空格
- 支持自定义替换

判断：适合工具页，但不属于视频处理。它更像现有「字幕校对 / 繁简转换」旁边的字幕文本批处理能力，已单独落地为「CC 字幕整理」页面。

## 推荐第一批

### 字幕工具：CC 字幕整理（已实现）

目标：把 Web CC 字幕整理成适合后续 Aegisub 精修的结构化 ASS 文本。

适合放在工具页的原因：

- 它不是通用字幕编辑，而是固定工作流的一键整理。
- 和现有「字幕校对」「繁简转换」一样，属于文本批处理。
- 能减少在 Aegisub 里重复运行 Lua 宏或手工拆行的步骤。

第一版可支持：

- 读取 `.ass` / `.ssa` 文本
- 对 Dialogue 行执行 `[说话人]台词` 拆分
- 自动设置样式名：`花字`、`听轴`
- 清理 `...`、`\N`、多余空格
- 可选执行自定义替换
- 输出新字幕文件，不覆盖原文件

暂不建议第一版支持：

- 任意成员字典编辑（已通过自定义替换词典支持）
- 任意样式名映射
- 复杂多标签 CC 行处理

### 视频工具候选

### A. 封装转换

目标：把视频快速转成 MP4 容器，默认不重编码。

候选命令：

```bash
ffmpeg -i input.ext -c:v copy -c:a copy output.mp4
```

注意点：

- 如果音频是 ADTS AAC，可能需要 `-bsf:a aac_adtstoasc`。
- 如果源流不兼容 MP4 容器，应给出错误提示，而不是自动重编码。

### B. 音视频分离

目标：从一个视频里提取音频或无声视频。

候选命令：

```bash
ffmpeg -i input.mp4 -vn -acodec copy output-A.m4a
ffmpeg -i input.mp4 -an -vcodec copy output-V.mp4
```

注意点：

- 音频输出扩展名应根据编码选择，AAC 用 `.m4a`，AC3 可用 `.ac3`。
- 视频无声输出优先保持原容器或输出 MP4。

### C. 音视频合并

目标：将一个视频文件和一个音频文件合并为 MP4。

候选命令：

```bash
ffmpeg -i audio.m4a -i video.mp4 -acodec copy -vcodec copy output_merge.mp4 -y
```

注意点：

- 应自动识别哪个是视频、哪个是音频。
- 默认不重编码。
- 需要处理时长不一致的提示，可提供“按较短流结束”选项。

### D. TS / 分片合并转 MP4

目标：选择一个目录，将其中的 `.ts` 分片按顺序合并并转为 MP4。

脚本思路：

```bat
copy /b "%folder%\*.ts" "%output_name%.ts"
ffmpeg -i "%output_name%.ts" -vcodec copy -acodec copy "%output_name%.mp4"
```

产品化时建议改为 ffmpeg concat，不直接依赖 `copy /b`：

```bash
ffmpeg -f concat -safe 0 -i list.txt -c copy output.mp4
```

注意点：

- 文件排序必须可预览，避免 `1.ts, 10.ts, 2.ts` 这种顺序问题。
- 合并列表应写入临时目录，完成后清理。

## 暂缓项

- 快速转码工具页入口：已转为压制页 `视频处理`，不再作为工具页第一批候选。
- 无字幕水印独立入口：文字水印仍暂缓；图片 LOGO 已由压制页覆盖。
- 视频分割：按秒、按份、截取片段。
- 字幕格式转换：VTT 转 ASS 等。
- 修改 MD5。

## 建议结论

工具页建议按 `文字处理` 和 `媒体处理` 两个一级分组重构。第一版应保持边界清楚，先做高频且不和压制页/Aegisub 重叠的能力。

第一批：

| 分组 | 页面 | 功能 |
| --- | --- | --- |
| 文字处理 | 字幕校对 | 保留现有能力 |
| 文字处理 | 繁简转换 | 保留现有能力 |
| 文字处理 | CC 字幕整理 | Web CC 拆分、台词清理、自定义替换 |
| 媒体处理 | 封装转换 | 视频转 MP4、TS/分片合并转 MP4、添加视频封面、合并视频+音频 |
| 媒体处理 | 音视频处理 | 提取音频、提取无声视频 |

第二批：

| 分组 | 页面 | 功能 |
| --- | --- | --- |
| 媒体处理 | 视频分割 | 按秒切分、按份切分、截取片段 |

暂缓：

| 分组 | 功能 | 原因 |
| --- | --- | --- |
| 媒体处理 | 快速转码工具页入口：旋转、镜像、缩放、帧率、码率、反交错 | 已在压制页作为 `视频处理` 落地，工具页不重复做入口 |
| 媒体处理 | 水印：文字水印、图片 LOGO 水印 | 图片 LOGO 已在压制页落地；文字水印和独立无字幕水印入口暂缓 |
| 文字处理 | 字幕格式转换：VTT 转 ASS 等 | Aegisub 已覆盖 |
| 文字处理 | 修改 MD5 | 不适合作为主产品功能 |

这些功能和现有压制页、字幕校对、繁简转换的边界相对清楚，也最贴近现有脚本里的高频实用能力。
